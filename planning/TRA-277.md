# TRA-277: Global Navbar тАФ Consistent Navigation Across All Pages

## Goal

Replace the 3 separate navbar implementations with a single `GlobalNavbar` component that appears on every page. Full-width at the top of the page, animates to a ~75% width floating pill shape on scroll.

## Design (inspired by wing.com)

### Scroll behavior
- **At top (scrollY = 0):** Full-width bar, flush with viewport edges, transparent/subtle background
- **On scroll (scrollY > 40):** Shrinks to ~75% width, centered, rounded-full pill shape, backdrop blur + subtle shadow. Smooth 500ms ease-out transition.

### Layout
- **Left:** TRAVYL logo + paper plane icon тЖТ links to `/`
- **Center:** Nav links тАФ Discover (`/`), Places (`/places`), Trips (`/trips`), Profile (`/profile` тАФ auth only)
- **Right:** Avatar dropdown (auth) or "Log In" button (unauth)

### Consistent everywhere
- Same component renders on all route groups: `(main)`, `(dashboard)`, `(trips-app)`
- Trip pages keep their `TripNavbar` as a **sub-nav** below the global nav
- Auth pages (login/signup) show a minimal version (logo + back only, or hidden)

### Avatar dropdown (unchanged)
- User info, Profile link, Settings link, Theme toggle, Sign out

### Color adaptation
- On homepage hero (dark background): white text, transparent bg
- Everywhere else: dark text (#1e3a5f), white/blur bg
- Active link: filled pill with brand color

## Implementation plan

### Step 1: Create `GlobalNavbar` component
- New file: `apps/web/components/GlobalNavbar.tsx`
- Port logic from `navbar.tsx` (it's already closest to the target)
- Key change: width transitions from `w-full` тЖТ `max-w-5xl` (roughly 75%) on scroll
- Remove the floating pill at rest тАФ starts as full-width flush bar
- On scroll: add rounded-full, margin, max-width constraint, backdrop-blur, shadow

### Step 2: Mount in root-level layout
- Add `GlobalNavbar` to `apps/web/components/providers.tsx` (or a new layout wrapper)
- Remove `<Navbar />` from `apps/web/app/(main)/layout.tsx`
- Remove `<DashboardNavbar />` from `apps/web/app/(dashboard)/layout.tsx`
- Adjust `pt-` padding on each layout's `<main>` to account for the global nav height

### Step 3: Adjust trip pages
- Trip pages currently have no global nav (just TripNavbar)
- The global nav will now appear above TripNavbar
- May need to adjust `trip-layout-inner.tsx` top padding

### Step 4: Delete dead code
- Delete `apps/web/components/DashboardNavbar.tsx`
- The old `navbar.tsx` becomes the new `GlobalNavbar.tsx` (rename or replace)

### Step 5: Hide on auth pages
- On `/login` and `/signup`, either hide the navbar or show minimal version (just logo)
