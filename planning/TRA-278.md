# TRA-278: Image Optimization — Slow Page Load Times

## Goal
Fix slow page load times caused by unoptimized images: convert raw `<img>` to `next/image`, add cache headers, preconnect hints, webp format, lazy loading, and error handling.

## Linear
https://linear.app/travyl/issue/TRA-278

## Streams

### Stream 1: API Routes — Cache Headers + WebP
- Add `Cache-Control: public, s-maxage=86400, stale-while-revalidate=604800` to `/api/images` and `/api/destination-image`
- Add `&fm=webp&q=75` to Unsplash URLs in `/api/images` and fallbacks in `/api/places`

### Stream 2: Layout — Preconnect Hints
- Add `<link rel="preconnect">` for `images.unsplash.com`
- Add `<link rel="dns-prefetch">` for `flagcdn.com`, `i.pravatar.cc`

### Stream 3: Hero Carousel — Lazy Loading
- Add `loading="lazy"` to non-visible hero slides in `(main)/page.tsx`
- Only first slide should be eager

### Stream 4: Convert High-Impact Components to next/image
- GetInspired.tsx — raw `<img>` → `<Image>` with fill + referrerPolicy
- TripMagazineHero.tsx — raw `<img>` → `<Image>` with fill + sizes
- TripListItem.tsx — raw `<img>` → `<Image>` with fixed dims
- SpotlightPreview.tsx — raw `<img>` → `<Image>` with fill + sizes
- HotelCard.tsx — raw `<img>` → `<Image>` with fill

### Stream 5: Error Handling
- Add `onError` + `imgError` state pattern to components missing it
- Priority: TripCard, TripListItem, HotelCard, SpotlightPreview, SpotlightResultItem, navbar avatars

## Known Issues
- CorkBoardMap.tsx uses template string `<img>` — out of scope (needs refactor)
- CSS backgroundImage in EventBlock/CalendarView — out of scope for now
